# sari-sari-utang-tracker
 Web-based utang tracking system for sari-sari stores using Flask and SQLite

## Render deployment

This repo includes `render.yaml` for a Render web service.

Render settings:

- Build command: `pip install -r requirements.txt`
- Start command: `gunicorn app:app`
- Health check path: `/healthz`
- Database: Render Postgres via `DATABASE_URL`

The app uses `DATABASE_URL` when it is available, so Render connects to the managed Postgres database defined in `render.yaml`. For local development without `DATABASE_URL`, it falls back to SQLite at `database.db`.

Set `USER_NAME` and `PASSWORD` in the Render dashboard when prompted. `SECRET_KEY` is generated by Render.

If you already have live data in SQLite, export or migrate it before switching the deployed service to Postgres. The new Postgres database starts empty except for the default products inserted by the app.
